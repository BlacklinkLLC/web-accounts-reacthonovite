Password Requirements

To maintain security, all passwords must meet these requirements:

  • Minimum length: 8 characters
  • Complexity: At least one uppercase letter
  • Special characters: At least one number or symbol
  • Validation: Real-time password strength indicator

Users can reset their password anytime using the "Forgot Password" feature. A secure reset link will be sent to the registered email address.

Common Issues

Invalid email domain error:

  • Verify your email matches your district's approved domain
  • Check for typos in your email address
  • Contact your school administrator to confirm domain whitelist
  • Try using your school's Google Workspace account

Google Sign-In not working:

  • Ensure you're using an approved Google account
  • Clear browser cookies and cache
  • Try a different browser or incognito mode
  • Check that pop-ups are enabled for blacklink.net
  • Verify your network doesn't block Google OAuth
üìß

Password & Account Recovery

Forgot password link not received:

  • Check your spam/junk folder
  • Wait 5-10 minutes for email delivery
  • Verify the email address is correct
  • Add noreply@blacklink.net to your contacts
  • Request a new reset link if needed

Account locked or suspended:

  • Too many failed login attempts (auto-unlock after 30 minutes)
  • Violation of Terms of Service
  • Contact support at contact@blacklink.net
  • Include your username and registered email

Sync & Data Issues

Data not syncing across devices:

  • Check your internet connection
  • Sign out and sign back in
  • Verify you're using the same account on all devices
  • Clear app cache (Settings ‚Üí Storage ‚Üí Clear Cache)
  • Update to the latest version of the app

Missing data or settings:

  • Allow 2-3 minutes for cloud sync to complete
  • Check Firestore connection status in settings
  • Ensure you're not in offline mode
  • Contact support if data loss persists

Firebase Authentication API

Blacklink uses Firebase Authentication SDK for all account operations:

Initialize Firebase:

import { initializeApp } from 'firebase/app';
import { getAuth } from 'firebase/auth';

Sign Up with Email:

createUserWithEmailAndPassword(auth, email, password)

Sign In with Google:

signInWithPopup(auth, googleProvider)

Password Reset:

sendPasswordResetEmail(auth, email)

Firestore Data Structure

User data is stored in Firestore with the following structure:

Users Collection:

users/{userId}/
  displayName: string
  email: string
  photoURL: string
  role: "student" | "teacher" | "admin"
  district: string
  createdAt: timestamp
  lastLogin: timestamp

Security Rules Example:

match /users/{userId} {
  allow read, write: if request.auth.uid == userId;
}

Integration Examples

Quick examples for common integration scenarios:

Get Current User:

const user = auth.currentUser;
if (user) {
  console.log(user.email, user.displayName);
}

Listen to Auth State:

onAuthStateChanged(auth, (user) => {
  if (user) {
    // User is signed in
  } else {
    // User is signed out
  }
});

Update Profile:

updateProfile(auth.currentUser, {
  displayName: "New Name",
  photoURL: "https://example.com/photo.jpg"
